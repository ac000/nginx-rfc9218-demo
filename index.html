RFC9218 HTTP/2 Priority Demo

This demo shows RFC9218 Extensible Prioritization in action. Resources are requested simultaneously but with different priority levels:

  • u=1 (High) - Critical resources that should load first
  • u=3 (Medium) - Default priority
  • u=6 (Low) - Background resources that can wait

With RFC9218, nginx prioritizes sending high-urgency resources before low-urgency ones, even when they were requested at the same time.
